# AI协作中心（Swarm Studio）主操作界面设计

**日期**：2026-06-22
**状态**：待评审
**高保真原型**：`.superpowers/brainstorm/12551-1782031991/content/cockpit-final-polished.html`
**适用项目**：hermes-studio（`upstream/hermes-studio`），样式系统 Pure Ink（`styles/variables.scss`）

---

## 1. 背景与目标

当前应用登录后直接进入聊天页，左侧是固定的 `AppSidebar`，缺少一个统筹人机协作的总览工作台。本设计新增一个**主操作界面（驾驶舱）**，作为人机混合团队（人类协作伙伴 + Agent 集群 + 人机 1:1）的指挥中心。

**核心诉求**：

- 通过 Kanban 统筹人类协作伙伴的互动、本地 Agent 集群的群聊、本地人机 1:1 协作
- 既有突出重点的全貌概览，又可针对具体任务接入，进行补充/评估/决策/审批等互动
- 既不失焦又不空洞——总有丰富实时状态（不空洞），但始终有明确的当前焦点（不失焦）
- 具备时序事件流的感觉，体现时间、空间和组织层面的协作联动
- 遵循现有 Pure Ink 朴素风格，简洁、重点突出、有结构与逻辑层次

**不在本设计范围**：替换现有聊天/看板/监控等页面本身；这些页面作为完整视图仍可独立访问，驾驶舱通过协作入口与嵌入视图与之衔接。

## 2. 设计决策记录

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 范围 | 重构整体 Shell 布局，新增驾驶舱主页 | 三类工作（协作/管理/易用）需统一框架 |
| 形态 | 三段联动式（全貌→聚焦→处理），非侧栏导航 | 页面多、要同屏盯住协作+状态 |
| 视觉风格 | 严格 Pure Ink 黑白灰 | 收敛颜色过载，回归专业克制 |
| 注意力条强度 | 克制提醒条（浅底+左色条+文字） | 重心应是右栏工作区，而非顶部提醒 |
| 颜色维度 | 仅 status 用 error/warning/success；类别/阶段/优先级用灰度+文字+字重 | 单一颜色维度，避免信息过载 |
| A2UI 灵活度 | 模板+可定制折中（生成→编辑→锁定→复用） | 平衡灵活性与稳定性 |

## 3. 整体布局

三列 flex 布局（非 grid，便于折叠），自上而下：顶栏 → 注意力条 → 主体三列。

```
┌─────────────────────────────────────────────────────────┐
│ 顶栏：品牌 │ 日程 │ 时钟 │ 搜索 │ 在线状态 │ 通知 │ 用户 │
├───────────┬─────────────────────────────────────────────┤
│ 注意力条  │ 3 需要你  │ [待办项…] │ 历史                 │
├───────────┼───────────────┬─────────────────────────────┤
│           │  面包屑+任务  │  模式切换：⚡工作项 💬协作 ⌘编程│
│  左栏     ├───────────────┼─────────────────────────────┤
│  Kanban   │   协作图      │                             │
│  统筹     │   (图谱画布)  │      右栏·工作区（重心）     │
│           ├───────────────┤   ┌─────────┬───────────┐   │
│  ·筛选    │   时序事件流  │   │ A2UI表单 │ 文件资源  │   │
│  ·三类    │   (纵向时间线)│   │         │ 管理器    │   │
│  ·按优先级│               │   └─────────┴───────────┘   │
│           │               │      底部：Claude Code/稍后/提交│
└───────────┴───────────────┴─────────────────────────────┘
```

**列宽比例**：左栏 220px（固定）· 中栏 340px（固定，可收窄）· 右栏 flex:1（弹性，重心）。右栏视觉权重最高（字号、间距、内容密度最大）。

**间距体系**：统一 8 倍数——4/8/12/16/24/32。圆角三档：6px(小) / 8px(中) / 12px(大)。

## 4. 视觉设计规范

严格遵循 Pure Ink（`styles/variables.scss`）：

### 4.1 配色（仅黑白灰 + 三个 status 色）

| 用途 | 变量 | 浅色值 |
|------|------|--------|
| 背景 | bg / bg-2 / bg-sb / bg-card | #fafafa / #f0f0f0 / #f5f5f5 / #fff |
| 边框 | bd / bd-l | #e0e0e0 / #ebebeb |
| 文字 | k0 / k1 / k2 / k3 / k4 | #1a1a1a / #333 / #555 / #888 / #bbb |
| 强调 | accent | #333 |
| 状态 | error / warning / ok | #c62828 / #e8a838 / #2e7d32 |

**颜色使用纪律**：status 三色仅用于"需要用户注意"的语义点（阻塞、风险、待办标记、已办完成、注意力条严重度色条）。绝不用于装饰。

### 4.2 非颜色维度的区分（替代彩色）

| 维度 | 区分方式 |
|------|----------|
| 工作流类别 | 分类标题前的线条样式：实线粗(k0) / 实线细(k3) / 点线(k4) |
| 任务优先级 | 字重：P0 加粗 k0 / P1 半粗 / P2 常规；右上角 P0-P3 文字标记 |
| 任务状态 | 文字标签 + 灰底；仅"阻塞"用淡红、"待审"加粗，其余纯灰 |
| 选中态 | 全局统一：左侧 3px 黑色色条 + 浅灰底(rgba(0,0,0,.05)) |

### 4.3 选中态统一语言

**所有可联动选中的元素**（Kanban 任务、拓扑节点、时序节点、文件节点、历史项）使用同一选中样式：左侧 3px 实色条 + `rgba(0,0,0,.05)` 浅底 + 黑色文字。这是用户形成"点哪里→哪里亮→联动哪里"肌肉记忆的关键。

## 5. 各区域详细设计

### 5.1 顶栏（topbar，高 44px）

从左到右：

1. **品牌**：`■ AI协作中心  Swarm Studio`（mark 方块 + 主名加粗 + 副名细字）
2. **日程按钮** `📅 日程 [n]`：点击弹出日程面板（见 5.2）
3. **系统时钟**：`2026-06-22 周一  14:36:28`，每秒刷新，`tabular-nums` 等宽不跳
4. **搜索** `搜索任务、空间、Agent ⌘K`：全局命令面板入口
5. **在线状态**：Agent 头像组（灰底）+ 人数 + 人类头像组 + 人数
6. **通知按钮** `通知 [n]`：红色角标=未读数，点击复用历史弹窗样式
7. **当前用户** `头像 姓名 ▾`：点击弹出用户菜单

**日程弹窗**：月历（今天高亮、有事件日期带点）+ 选日期显示该日事件流 +「+ 待办」新增提醒。

**用户菜单**：头像+邮箱头部 → 设置 / 账户资料 / 主题与外观 / 连接与集成 / 退出登录。各项跳转 `/settings`。

### 5.2 注意力条（克制提醒）

浅灰底，分三段：

- **左侧标签**：`[n] 需要你`（n 为数字徽章，黑底白字）
- **中部待办项**：横向滚动条目，每项 = 左色条（严重度：红=紧急/黄=中等/灰=常规）+ 文字 + `→` 箭头
- **右侧历史按钮** `🕘 历史`：点击打开历史回溯弹窗（见 5.8）

点击任一待办项 → `gotoWork()`：选中左栏对应任务 + 切右栏到工作项模式 + 更新待办描述。**点击即直达处理，不再二次跳转。**

### 5.3 左栏 — Kanban 统筹（220px，可折叠）

**头部**：`Kanban 统筹` + `↓ 优先级`（排序标识）。

**筛选器**（三行 chip 组）：
- 优先级：P0 / P1 / P2 / P3
- 状态：待审 / 阻塞 / 进行中 / 待办 / 完成
- 类别：人类协作 / Agent集群 / 人机1:1

筛选 chip 选中=黑底白字，未选=描边灰字。多选叠加。

**任务列表**：
- 按优先级降序排列（重要的在前），同优先级内按更新时间
- 三个分类分区，每区标题带类别线条样式 + 任务计数
- 任务卡：左侧选中色条（选中时显示）+ 右上角优先级标记 + 标题（字重随优先级）+ 状态标签 + 负责人

每个任务携带 `data-ws` 属性（其对应的 Workspace 路径），选中时联动中栏面包屑、右栏文件树根目录、Claude Code 终端根目录。

**折叠**：点边缘 ◀ 按钮 → 收为 32px 竖条（竖排"KANBAN"文字），点竖条恢复。

### 5.4 中栏 — 协作图 + 时序流（340px，可折叠）

**头部**：面包屑 `项目 › 任务名` + 状态 chip。任务名随左栏选中动态更新。

#### 协作图（图谱画布）

- 标题栏：`协作图` + 层级切换段组（项目级 / 需求级 / 应用级）
- 画布：网格点背景，节点可鼠标拖拽，连线表示关联
- 节点选中态：黑边框 + 左侧色条（统一语言）
- **多层级拓扑**：切换层级显示不同维度的关联关系——项目级（需求树）/ 需求级（子需求+任务）/ 应用级（代码文件+编排链，默认）
- 点击拓扑节点 → 下方时序流切换为该事项的事件

**空间-组织联动**：应用级视图体现"谁在哪个空间、谁和谁怎么联动"——节点显示在场者，连线标注 A2H（Agent↔人）/ A2A（Agent↔Agent）关系。

#### 时序事件流

- 标题：`时序流` + 来源标签（当前聚焦的事项，随拓扑选择更新）
- **折叠历史**：较早的事件默认折叠为「▸ N 条较早历史」，点击展开
- 纵向时间线：每个事件 = 头像 + 主体名 + A2A/A2H 标签 + 时间 + 内容 + 待办/已办状态
- **当前节点突出**：选中节点用统一选中态（黑点 + 浅底 + 加粗）
- 风险节点：时间线圆点用 error 红色
- 点击时序节点 → 右栏切换到该节点的处理内容

**三层联动**：每个事件标注 时间·空间·组织 三层维度，事件间显式标注联动（如 review-agent 发现风险 → 触发 qa-agent 补用例，用嵌套小条串起）。

### 5.5 右栏 — 工作区（flex 弹性，重心）

#### 模式切换器（统一入口）

顶部 tab 切换右栏的三种工作模式：
- **⚡ 工作项**：A2UI 表单 + 文件资源管理器（默认）
- **💬 协作**：嵌入式聊天视图（角标显示活跃频道数）
- **⌘ 编程**：Claude Code 终端

右侧有 `⛶ 最大化` 按钮：点击后中栏自动折叠，右栏占满全宽；再点还原。

#### 协作入口条

模式切换器下方，显示当前事项**已关联的协作频道**（chip 形式）：
- `👥 auth-svc 联调 3人`（Matrix）
- `💬 review-agent`（单聊）
- 点任一 chip → 切到协作模式进入对应聊天

右侧 **`+ 新增协作`** 按钮：点击弹出菜单（Matrix 聊天室 / Agent 单聊 / 群聊），选择类型→邀请成员→与当前事项绑定→自动出现在协作入口。

#### 工作项模式（水平分栏）

左：**A2UI 处理表单**
- 待办 banner（黄底，严重度标记）
- 模板复用提示（基于 X 模板生成，N 处不同）
- 代码 diff 块
- 表单字段（结论单选+推荐标注 / 风险标签 chips / 评审意见文本框）
- 处理说明
- 底部按钮：`⌘ Claude Code` / `稍后处理` / `提交决定`（其中「⌘ Claude Code」是编程模式的快捷入口，等同于点模式切换器的「⌘ 编程」tab）

右：**文件资源管理器**（230px）
- 标题：`任务文件` + `当前任务的 Workspace`
- 根目录路径（随 Kanban 任务 `data-ws` 动态切换）
- 文件筛选框
- 树形结构：目录可展开/折叠，文件可选中（统一选中态）
- 本次修改的文件标 `M` 徽章 + 加粗

#### 协作模式（嵌入式聊天）

头部：频道名 + 成员列表 + 返回工作项。
消息流：区分人/Agent/你的头像与气泡（你的消息黑底白字右对齐）。
输入框：`@提及人或 Agent…` + 发送。

对应真实路由：Matrix→matrix 聊天室；单聊→`hermes.session`(sessionId/profile)；群聊→`hermes.groupChatRoom`(roomId)。

#### 编程模式（Claude Code 终端）

深色终端，以当前任务的 Workspace 为根目录的 sandbox 编程模式：
- 头部：macOS 圆点 + 标题 + 根目录路径 + `sandbox` 标签 + 退出
- 预填任务上下文（任务名、关联文件、review-agent 的风险提示与建议）
- 输入行：`❯` 提示符，Enter 发送指令，sandbox 限定读写范围
- 根目录随任务切换

### 5.6 A2UI 生命周期（模板化折中）

右栏工作项的表单由 Agent 生成，遵循"生成→编辑→锁定→复用"流水线：

1. **生成**：Agent 按任务上下文自由生成 UI 形态（审核单/对比卡/表单…），预填字段值、标注推荐项
2. **编辑调整**：用户字段级编辑（hover 出工具条：✎改/⇅排序/✕删），可从 8 类受控字段类型添加（文本/单选/多选/打分/日期/关联人/文件/关联任务）
3. **锁定为模板**：满意后一键锁定，命名存入模板库，带匹配条件与引用次数
4. **复用·调整或接受**：同类任务参考模板生成，顶部提示"基于模板 X，N 处不同"，用户可接受或再次调整

**折中原则**：灵活性来自生成+编辑+复用可调；稳定性来自字段类型受控+锁定模板+同类默认参考。

### 5.7 三列折叠机制

每列均可独立最大化/最小化：
- 点列边缘的折叠按钮 → 列收为 32px 贴边竖条（竖排标题），点竖条恢复
- 内容隐藏（`display:none`），只留 rail
- 右栏额外有最大化按钮（折叠中栏，右栏占满）

满足不同工作场景：专注处理（左中折叠）/ 全面概览（全展开）/ 深度编程（中栏折叠+右栏最大化）。

### 5.8 历史回溯弹窗

点击注意力条「🕘 历史」打开。

- **筛选器**（五行 chip）：搜索 / 时间（今天·本周·本月）/ 类别 / 动作（审批·决策·补充·评估·委派）/ 状态（进行中·已完成·已归档）
- **事件列表**：每条 = 时间 + 状态色点 + 标题 + 标签，hover 显示左色条（统一选中语言）
- **点击事件 → 关闭弹窗，筛选下方三栏**为该事件相关内容（左栏选中任务、中栏时序切换、右栏更新标题与待办）
- **归档事件**：灰色调 + `已归档` 标签 + 标题置灰，进入后全局 `archived` 只读态

**归档只读态**：
- 全局灰色调：任务变淡、待办条变灰、表单控件半透明禁用
- 右栏顶部显示 `已归档 · 只读` 徽章
- 底部操作按钮隐藏，仅保留 `+ 发起新关联协作`（可针对归档事项拉起新 Matrix/单聊/群聊）
- 点回非归档事件恢复正常可编辑

## 6. 交互动线（核心联动逻辑）

一条贯穿全局的联动链：

```
注意力条待办项 ─┐
Kanban 任务 ────┼──→ selTask()
                │     ├─ 左栏：高亮任务（统一选中态）
                │     ├─ 中栏：面包屑任务名更新
                │     ├─ 右栏：文件树根目录 = 任务 data-ws
                │     ├─ 右栏：Claude Code 根目录同步
                │     └─ 右栏：切到工作项模式
                │
协作图拓扑节点 ─→ 中栏时序流切换为该事项事件
                │
时序事件节点 ───→ setMode('work') + 右栏 A2UI 切换为该节点处理
                │
右栏模式切换 ───→ 工作项表单 / 嵌入聊天 / Claude Code 终端
                │
历史事件 ──────→ recall()：筛选三栏 + 归档只读态（如归档）
```

每一步的选中反馈都用统一的"左色条+浅底"语言，用户形成稳定的交互预期。

## 7. 与现有系统的集成

### 7.1 样式

直接使用 Pure Ink CSS 变量（`--bg-*`/`--border-*`/`--text-*`/`--success`/`--error`/`--warning`），自动支持 light/dark/comic 三主题。驾驶舱不引入任何自定义色值。

### 7.2 路由与页面

| 驾驶舱入口 | 现有页面/路由 |
|-----------|--------------|
| 协作·Matrix | matrix 聊天室 |
| 协作·Agent 单聊 | `hermes.session`（sessionId + profile query） |
| 协作·群聊 | `hermes.groupChatRoom`（roomId） |
| 用户菜单·设置 | `/hermes/settings` |
| 任务 | Kanban 任务（复用 kanban API 的 workspace/tenant/assignee） |

驾驶舱的协作入口与「↗ 打开」按钮可跳转完整的现有页面；嵌入视图是这些页面的精简内嵌版。

### 7.3 复用现有组件模型

- **Actor 模型**：group-chat 的 `RoomAgent`（Agent）/ `MemberInfo`（人）/ `ProfileAvatar`，用于协作图、时序流、聊天的头像与角色区分
- **任务模型**：Kanban 的 `workspace_kind`/`workspace_path`/`tenant`/`assignee`/`created_by`，用于空间-组织协作维度
- **事件模型**：Kanban 的 `KanbanEvent.kind`，用于时序事件流
- **注意力条**：复用 `KanbanAttentionStrip` 的严重度体系（⚠/!!/!!!）与 rgba 背景规范

## 8. 响应式与可访问性

- **断点 768px 以下**（移动）：三列改为单列堆叠 + 底部 tab 切换；注意力条待办项改为纵向列表
- **键盘**：⌘K 命令面板 · ⌘↵ 提交 · ⌘. 稍后 · 1/2/3 切换决定 · Tab 在表单字段间移动
- **可折叠**保证小屏可用：窄屏默认折叠左栏与文件树

2. **协作图多层级的数据来源**：项目级/需求级拓扑关系从何处获取？是否需要新建关联关系数据结构？
3. **Claude Code 终端的真实集成**：sandbox 编程模式的进程隔离、文件系统边界如何实现？是否复用现有的 coding-agent 机制？
4. **历史回溯的归档判定**：什么条件下事项自动归档？（如：关联任务完成 N 天后 / 手动归档）
